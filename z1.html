Homework1: For the three categories of web software technology we've just defined, you are asked to find out which category so-called "Chrome plug-in" technology belongs to. Please describe the basic idea of the technology, and give your reasoning on its position in the technology categories.

答：我认为Chrome plug-in属于分布式对象技术，正文课件中所提到的”The central idea of CORBA is the notion of object BUS, which allows objects to plug and play”，三个最重要的分布式对象技术之一的CORBA的中心思想是对象总线的概念(一个软件基础设施/中间件),它允许对象即插即用。这和谷歌插件的思想也是完全一致。 Chrome plug-in是当浏览器提供的功能已经不能满足用户的需求而产生的，所以 Chrome plug-in的作用就是使用户能够更方便的使用浏览器的一项功能。

Homework2: In a normal intercommunication between Web browser and Web Server, Web Server closes the network communication right after it sends back the document requested by the browser. Think about this statusless character of World Wide Web. Give some good points and bad points of it. Can a web server know exactly how many browsers are reading its document?

答：优点：在发送回浏览器所需要的资源后关闭连接，最大的好处是可以节约连接资源，满足更多用户的需求。因为资源有限，连接的总数是有限的。

    缺点：当关闭连接以后，如果用户需要重新发送请求，或者请求其他资源，则服务器需要重新新的一轮连接，会造成一些浪费。

Homework3(optional): Open Question: of those client side technologies, why Java Applet is getting dead and Javascript success? Try to explain from the "REST" perspective.

答：Applet 是Java刚流行的一种语言 ! 可以嵌套在浏览器中，但是它有很大的弊端，比如编写的时候必须有jdk环境 ，运行的时候浏览器必须有jre环境！这样就会使得极大部分用户无法使用applet,这些都极其不利于它的发展，后来flash等动画流行后，该技术逐渐被替代。现在基本已经很少有人用了。 而javascript越来越流行，HTML5现在应用越来越广泛，HTML5是运行在浏览器当中，而浏览器中的脚本语言就是javascript，所以大家没有其他选择，只有javascript才能在所有的浏览器中很好的运行，虽然有大公司开发了代替javascript的其他语言，但是应用的还非常的少，所以目前javascript很流行。而且JavaScript 易学难精，而现在各种 JS 框架，如 jQuery，YUI 又恰好弥补了这一点，初级开发人员甚至不需要真正理解 JavaScript 这门语言就可以轻易上手用 JavaScript 做项目 而从REST的角度，applet违反了REST的一系列约束，不满足REST设计风格,如提供服务的服务器和使用服务的客户需要被隔离对待。applet会在浏览器端分配过多的任务，不利于用户使用。
